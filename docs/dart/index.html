Dart

Tips​

assert動作の環境差異​

本番環境では無視される。開発環境では、条件が偽の場合に例外をスローする。

lateの初期化​

late String userName = _loadUserName();

上記のように、宣言と同時に初期化するようなコードの場合、userNameが最初に使われるタイミングで初期化（_loadUserName()が実行）される。

引数のデフォルト値の非推奨事項​

引数のデフォルト値を設定するために、古い記述のコードでは:（コロン）が使われている場合がある。現在は=が推奨されており、:は非サポートになる可能性がある。

戻り値を指定しない関数​

戻り値を指定しない関数はNULLを返却する。

foo() {}
assert(foo() == null);

型の検証​

object is Typeとobject.runtimeType == Typeの2パターンで型の検証が可能だが、前者（object is Type）の方がより安全に検証可能。

if (p is Point) {
  print('The type of p is Point');
}

Style​

  • class、enum、typedefなどはUpperCamelCaseを使う

  • 変数、定数、関数などはlowerCamelCaseを使う

  • ライブラリ、パッケージ、ディレクトリ、ファイル名などはlowercase_with_underscoresを使う

  • インポートプレフィックスはlowercase_with_underscoresを使う

    import 'dart:math' as math;
    import 'package:angular_components/angular_components' as angular_components;

  • 未使用のコールバックパラメータは_や__と定義する

    futureObj.then((_) {
      // callback
    })

linterで検知できないEffective Dart: Usage​

  • part ofを使う場合は相対パスで指定する
  • nullをbooleanに変換する場合は??を使う
  • 初期化されているか確認する必要があるならlateを使わない
  • nullableなフィールドをローカル変数にコピーして、型推論をさせる
  • 型を変更する必要がなければList.from()を使用しない
  • Genericsなどで済む場合はcast()を使わない
  • 可能な限りcast()を使わない
  • varとfinalに一貫性を持たせる
  • 計算できるものをメンバとして保持しない
  • コンストラクタに依存しないのであれば、宣言時に初期化する
  • コンストラクタの初期化リストで十分な場合は、lateを使用しない
  • catchしたエラーを適切に処理し、不用意に破棄しない
  • プログラム上のエラーの場合のみ、Errorオブジェクトをスローする
  • Futureをそのまま使わず、async/awaitを使って可読性を上げる
  • 有用な効果がない場合はasyncを使わない
  • streamを変換するために高階関数を利用する
  • Completerを直接使わない
  • FutureOr<T>の型を明確にするため、is Future<T>で検証する

linterで検知できないEffective Dart: Design​

  • 必要でない限り宣言はprivateにする
  • 1つのライブラリに複数のクラスを含めることを検討する
  • サブクラス化することを意図していない、もしくは不明なクラスの継承は避ける
  • クラスが継承可能か文書化する
  • インターフェースと意図できないクラスの実装を避ける
  • クラスがインターフェースとして使用できるかを文書化する
  • コンストラクタをconstにすることを検討する
  • フィールドにアクセスする操作は、getterを使用する
  • オーバーロードをごまかすために実行時の型テストの使用を避ける
  • 初期化子をもたないlate finalのフィールドを避ける
  • nullableなFuture、Stream、collectionを返さない
  • 関数宣言には戻り値の型を必ず記述する
  • 関数宣言の引数の型は必ず記述する
  • 推論されないGenericsの呼び出しは型を記述する
  • 推論可能なGenericsの呼び出しには型を記述しない
  • 不完全な汎用型を記述しない
  • 推論失敗ではなく、dynamicを明記する
  • 引数として関数を受け取る場合、受け取る関数の引数や戻り値を明記する
  • あらゆる型を受け取りたい時はdynamicではなく、Objectを検討する
  • 値を返さない非同期関数の戻り値はFuture<void>とする
  • 戻り値の型としてFutureOr<T>の使用を避ける
  • 引数を省略する可能性がある場合、名前付き引数にする
  • nullや空文字などを受け入れる必須パラメータは避ける
  • 範囲を指定する場合、開始と終了を含むパラメータを使用する
  • ==演算子は数学的な等価関係に従う

フローによる型推論にフィールドは適用されない​

次のコードはコンパイルエラーになる。

class Coffee {
  String? _temperature;

  void checkTemp() {
    if (_temperature != null) {
      print('Ready to serve ' + _temperature + '!');  // ★コンパイルエラー
    }
  }
}

これは次の2パターンで回避可能。

  • !でnullableでないことを明示する

    void checkTemp() {
      if (_temperature != null) {
        print('Ready to serve ' + _temperature! + '!');
      }
    }

  • ローカル変数に代入する

    void checkTemp() {
      final temperature = _temperature;
      if (temperature != null) {
        print('Ready to serve ' + temperature + '!');
      }
    }

乱数の利用​

Random()は暗号化目的には適さない、疑似ランダム機能を提供する。 よりセキュアな実装が求められる場合は、暗号的に安全な乱数生成器を用いているRandom.secure()の利用を検討する。
